1.2. Making the last registered user invisible
A security option that considered quite a few managers is a registration window in which the last notified user is no longer visible. Configure this policy via editing the Default Domain Policy.
Policy not activated.
Policy activated.
Right click on "Group Policy Management".
Right click on "Default Domain Policy" and Select "Edit".
Navigate to Security Options. Double-click Interactive Logon: Don't Display Last Signed-in.
"Define this policy setting" and activate "Enabled". The Explain tab says what the policy does exactly. Click "OK".
1.3. Centrally distribute a desktop wallpaper

We will set a Desktop wallpaper for the OU "herstellingen". This wallpaper will be placed on the Windows Server 2019.

Create a folder "wallpaper" on the server on the data partition and place a file (.png .jpg â€¦) in this folder. Share this folder and give the appropriate sharing permissions and folder permissions.

"Domain Users" full control on the share permissions.
"System" and "Administrators" were already in security permissions. We'll leave that one. "Add" "Domain Users" to it (use the suggested permissions).

Go to the group policy manager (see images):

- Specify the UNC path where the wallpaper is located.

- Server1is my server name, use the name of your server. Click "OK" when done.

- Right click in the group policy management on the OU fixes. Select Link an Existing GPO.

- Select the OU in question, and click "OK".

- Because it is a policy that is in the user configuration, we have to log out the user first, after logging back in, we see the wallpaper appear on the client computer.

- Log in to a workstation as a user that belongs to the OU.
