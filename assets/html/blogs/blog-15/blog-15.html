1. Step - Update Server
Update your package lists from the repositories. To do this, open a terminal. Now type the following commands in the terminals.

sudo apt-get update

sudo apt-get upgrade

2. Step - Install nano and Apache2
Now install nano and Apache2 (web server).

sudo apt-get install nano

sudo apt-get install apache2

3. Step - Configure Apache2
Type in the following command to delete the basic configuration files. You don't need this old configuration. We will create a new configuration file.

sudo rm -r /etc/apache2/sites-available/000-default.conf

sudo rm -r /etc/apache2/sites-available/default-ssl.conf

sudo rm -r /etc/apache2/sites-enabled/000-default.conf

sudo rm -r /var/www/html/
Now we will create a new configuration file for Apache. Type the following command in.

sudo nano /etc/apache2/sites-available/configurationfile.conf

In the new file you just created, type in the following lines. And then save it with CTRL + X.
Now we will create our location to put your website in. Use the following commands.

sudo mkdir /var/www/root

In this folder you will need to put your website information into. This is probably an index.html file. If you don't have a website and you want to learn how to make it, use the link.
4. Step - Restart Apache2 and enabled configurationfile.conf
Our initial configuration is done. Now, make sure you are restarting your Apache2 service to accept the changes that we had made so far. To do so, please type the following in your terminal.

sudo a2ensite (Name of configuration File)

sudo systemctl reload apache2

sudo service apache2 restart

This will restart the apache2 service with the new configuration that we made so far.
5. Step - Configure UFW (Ubuntu Firewall)
We'll check the firewall status to see if it's enabled. If so, we'll ensure that HTTP and HTTPS traffic is permitted so you won't run into firewall rules blocking you when it comes time to test.

sudo ufw status

If the firewall is not enabled then type.

sudo ufw enable

We'll need to open ports 80 and 443 for HTTP and HTTPS for later when we enable ssl. If you're working remotely, make sure the port for .

sudo ufw allow 80/tcp

sudo ufw allow 443/tcp

6. Step - Configure SSL & let's encrypt
Of course, it's also prudent to encrypt your connections. We will do this with let's encrypt. What is let's encrypt let's encrypt is a script that will allow us to automate our ssl configuration. Type the following command to install the script.

sudo apt-get install certbot python3-certbot-apache

Now we will run the script with following command.

sudo certbot --apache

The script will ask for some information.


The first question will ask for your email address. Please type your email address.
The second question. Will ask you to agree to the terms of service. Please type "Agree".
The third question will ask you if they can send you emails. Please type "NO".
The fourth question will ask you which website you want to encrypt. You will need to type in your domain name.
The last question will ask you to divert all HTTP traffic to the HTTPS sides you will need to answer yes by typing two.
